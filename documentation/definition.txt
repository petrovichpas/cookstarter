“CookStarter”


Сервис для подачи удаленных заявок на приготовление блюд в организациях общепита — кафе, рестораны, пиццерии. Везде где необходимо ожидать приготовления заказа.

Цель сервиса — экономия времени клиента на ожидание приготовления заказа. Для тех кто не любит ждать приготовления пищи и общаться с персоналом заведения.

Примеры:

1. Компания деловых людей отправляются провести переговоры в ресторан, по дороге делают заказ, по приезду подают блюда.

2. Студент GeekBrains торопиться с работы домой на вебинар, по дороге заказывает «пепперони» в любимой пиццерии (что расположена по пути домой) на вынос. Проходя мимо забирает заказ.



Сервис можно разделить на три части:

1. Сторона клиента — веб сайт, мобильное приложение (у нас про Java, будем делать под Android), телеграмм бот.

2. Сторона общепита — веб сайт, мобильное приложение, телеграмм бот, API для программных компонентов организации общепита.

3. Сторона сервера – spring boot приложение.



Типовой сценарий — клиент заходит в систему под своим аккаунтом(возможно привязать к номеру телефона), находит ресторан, кликает по блюдам, устанавливает количество, подтверждает заказ, на стороне ресторана приходит заказ(возможно уже сразу оплаченный), менеджер ресторана передает заказ на кухню.



Для MVP начнем с простой схемы — веб сайт (клиентский аккаунт), сервер, веб сайт(аккаунт администратора ресторана). Подготовка андроид.



Основные задачи для MVP:

1. Определиться с архитектурой.

2. Сверстать странички для клиента и ресторана.

3. Реализовать логику авторизации клиента.(по развитию проекта нужно будет добавить регистрацию)

4. Реализовать логику авторизации ресторана.(по развитию проекта нужно будет добавить регистрацию)

5. Реализовать логику добавления и редактирования меню ресторана.

6. Реализовать контекстный поиск ресторана.

7. Сверстать странички отображения клиента, ресторана(и меню) и заказа.

8. Реализовать логику отправки заказа клиентом и приема его рестораном.


Фичи — отложенный заказ, платежная система, избранное, рейтинг ресторана/клиента, пуш от ресторанов к клиентам по акциям, таймер для клиента о готовности, боты, соцсети, выбор столика.


Развитие — реализовать возможность взаимодействия с системой через остальные каналы кроме веб сайта, реализовать фичи.
